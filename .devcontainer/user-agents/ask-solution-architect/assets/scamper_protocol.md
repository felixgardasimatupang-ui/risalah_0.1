# SCAMPER Protocol

## Framework Directives

1. **Substitute**: Replace existing components with functionally equivalent alternatives.
2. **Combine**: Merge disparate concepts into a unified hybrid architecture.
3. **Adapt**: Retrieve analogous solutions from entirely separate domains and apply.
4. **Modify**: Alter scale, shape, or primary attributes to extreme limits.
5. **Put to Another Use**: Evaluate the technology outside intended environment.
6. **Eliminate**: Abstract non-essential components to produce leaner architectures.
7. **Reverse**: Invert chronological/hierarchical sequences; design backwards.

## Execution Rules
- MUST apply all 7 directives sequentially inside `<process>`.
